# mn-page-view 页面视图

组件名：mn-page-view 代码块： mn-page-view

# 安装方式

通过npm方式使用mn-mall-ui组件 本组件符合easycom (opens new window)规范，HBuilderX 2.5.5起，只需将本组件导入项目，并配置 easycom 规则后，在页面template中即可直接使用，无需在页面中import和注册components。

# 基本用法

在 template 中使用组件

    <mn-page-view sate="loading" ></mn-page-view>

# API

# 属性

属性名 类型 默认值 可选值 说明
state String loading loading:加载状态
loadFinish: 加载完成
netError: 网络错误
error:错误状态
empty: 空数据提示		 页面显示状态
tipMsg String 空 - 提示消息, 只有netError,error,empty 这三种状态才有效
showNavBar Boolean true true/false 是否显示navbar导航栏
navBarConfig Object { logoUrl:'', indexPageUrl:'/pages/index/index', navParams:{ navbar: 1, return: 0, title: '首页' }} - 导航栏属性配置
backTop Boolean false true/false 是否使用置顶功能（结合mn-back-top使用）为true时，内容的是通过scroll-view包裹进行一个滑动的监听
scrollTop Number 0 - 滑动位置，只有backTop为True时有效

# navBarConfig 数据结构

属性名 说明
logoUrl logo图标
indexPageUrl 首页地址，可以不传 默认 /pages/index/index
navParams 这个字段配置，和mn-navbar的paramter数据结构保持一致

eg

{
	logoUrl:'',
	indexPageUrl:'/pages/index/index',
	navParams:{
		navbar: 1, //自定义导航栏类型 0，带搜索；1正常导航
		return: 0,
		title: '首页',
	}
}

# 事件

事件名 事件说明 返回参数
onRetry 错误重试，只有state=error/netError,才有效 无
onScroll 内容区域滑动监听 scrollTop 顶部滑动距离，只有backTop为true时有效

# 插槽

名称 事说明
loading loading插槽
empty 页面空数据提示插槽
netError 网路异常，插槽
error 错误提示插槽

# 组件示例

	<template>
		<mn-page-view :state.sync="pageState" :tipMsg="pageTiprMsg" @onRetry="onRetry">
			<view>
				demo
			</view>
		</mn-page-view>
	</template>

	<script>
		export default {
			data() {
				return {
					title: 'Hello',
					pageState: 'loading',
					pageTiprMsg:'',
				}
			},

			onLoad() {

			},
			methods: {
				onRetry(){
					this.pageState="empty"
					this.pageTiprMsg="暂无数据"
				}
			}
		}
	</script>

# 注意

当state值为error/netError时，点击重试，同步页面为loading状态，需要为state添加同步修饰符.sync 即state.sync=xx,这样点击会自动同步为loading状态
